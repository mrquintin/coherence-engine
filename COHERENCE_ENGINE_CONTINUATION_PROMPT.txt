Deterministic ontology entity + relation extractor landed. This provides

structural input the hybrid domain step (prompt 06) will consume. The
extractor is lexical and POS-free — no ML, no NLP library.

- `core/types.py`: added frozen `Entity` (id, type, surface_forms,
  mentions), `OntologyEdge` (src, dst, relation, evidence_count), and
  `OntologyGraph` (entities, edges, schema_version). Existing dataclasses
  are untouched.
- `server/fund/data/ontology_lexicon.json` (NEW): actor_markers,
  causal_markers, metric_markers, competition_markers, plus the
  dependency/part/measurement/object marker lists and founder_forms used
  by the extractor.
- `domain/ontology.py` (NEW):
  - `extract_ontology(propositions) -> OntologyGraph`. Empty input →
    empty graph with `schema_version="ontology-v1"`.
  - Entity detection uses word-boundary substring search with longest-
    match non-overlap resolution (so "our team" beats "team").
  - Entity IDs: `sha256("{type}|{canonical}")[:16]`.
  - Surface forms are deduped case-insensitively; `mentions` is a raw
    occurrence count across all propositions.
  - Relation extraction walks each sentence: for every marker span it
    takes the nearest entity on the left and on the right and emits an
    edge (`causes` for causal markers, `depends_on` for dependency
    markers, `part_of` for part markers, `measures` for measurement
    markers). Every causal marker occurrence also registers a
    `causal_event` entity whose `mentions` doubles as the reason_count.
    Ambiguous causal markers (no entity on one or both sides) still
    register the event entity but do NOT emit an edge.
  - Competition handling: for each competition marker, prefer linking a
    founder actor to a non-founder actor with `competes_with`; if none
    of that pairing is available, link the two actor entities nearest to
    the marker span.
  - Output is fully deterministic: entities sorted by (type, id); edges
    sorted by (src, dst, relation); surface_forms sorted
    lexicographically by lowercase key.
- `domain/__init__.py`: re-exports `extract_ontology` and
  `OntologyGraph`.
- `tests/fixtures/ontology/sample_propositions.json` (NEW): 12-
  proposition fixture exercising every entity type (actor, metric,
  object, causal_event) and every relation type (causes, depends_on,
  part_of, measures, competes_with).
- `tests/test_ontology_extractor.py` (13 tests): empty input, byte-
  identical determinism via `json.dumps(..., sort_keys=True)`, at least
  one `causes` edge on fixture, every entity type exercised, every

  surface-form dedup, founder→named-competitor preference, ambiguous-
  marker causal_event without edge, package re-export identity,
  evidence_count aggregation across propositions, frozen dataclass
  instances, and a source-level check that spacy/nltk/transformers/
  torch/sklearn are not imported by `domain/ontology.py`.

by prompt 06).

Workflow orchestrator skeleton landed. The fund backend now ships a thin,
in-process, idempotent orchestrator that composes the nine pipeline stages
(intake -> transcript_quality -> compile -> ontology -> domain_mix -> score
-> decide -> artifact -> notify) and writes a checkpoint row per stage so
a failed run can resume at the first non-succeeded step. The orchestrator
WRAPS (does not replace) existing stage implementations: same
`ScoringService`, same `DecisionPolicyService`, same `persist_decision_artifact`,
same notification dispatch, and the same canonical event order
(`InterviewCompleted -> ArgumentCompiled -> CoherenceScored -> DecisionIssued`).

- `server/fund/services/workflow.py` (NEW):
  * `STEPS` — ordered 9-tuple of `(name, fn)` pairs:
    `intake`, `transcript_quality`, `compile`, `ontology`,
    `domain_mix`, `score`, `decide`, `artifact`, `notify`.
  * `class WorkflowError(Exception)` — validation / orchestration
    failures (missing app, unknown step); `class WorkflowResumeRefused`
    subclass specifically for `input_digest` drift on a succeeded step.
  * `canonicalize(value)` + `compute_digest(value)` — canonical JSON
    (sorted dict keys, non-JSON scalars coerced to str) -> SHA-256.
    Powers both `input_digest` (tamper detection on resume) and
    `output_digest` (stage-output fingerprint).
  * `WorkflowContext` dataclass — threads session, application_id,
    trace_id, idempotency_prefix, notification backend + dry-run dir
    through the stages; `app`, `score`, `decision_payload`,
    `canonical_decision`, `session_id`, `transcript_uri` are populated
    lazily via `_ensure_app` / `_ensure_score` / `_ensure_decision_payload`
    so a resumed stage can reconstruct upstream state without re-
    executing any already-succeeded side effects.
  * `run_workflow(session, application_id, *, resume=False,
                  force=False, trace_id="", idempotency_prefix="",
                  scoring_job_id="", notification_backend=None,
                  notification_dry_run_dir=None) -> WorkflowRun`:
    - Loads (resume) or creates (fresh) a WorkflowRun row.
    - For each stage: computes `input_digest` from the stage's
      declared inputs; if an existing WorkflowStep row is `succeeded`
      and input_digest matches, skips; if it diverges and `force=False`,
      raises `WorkflowResumeRefused` with `input_digest_mismatch:<stage>`;
      else (re)creates the step row in `running`, executes the stage
      body, computes `output_digest`, marks `succeeded` + finished_at.
    - On exception inside a stage body: marks the step `failed` with
      an operator-readable `error` string (no stack traces, no
      credentials), marks the run `failed` + `finished_at`, and
      re-raises the original exception unchanged.
    - Stage side effects are idempotent at the service layer (scoring
      is deterministic, `upsert_decision` is an UPSERT,
      `persist_decision_artifact` is digest-keyed, notification
      dispatch is keyed on `(application_id, template_id)`), so a
      re-run produces the same persisted state without duplication.
    - Event publishes use a stable, application-scoped
      `idempotency_prefix` (default `f"workflow:{application_id}"`)
      so outbox keys are deterministic across runs.

Hybrid domain reconstruction. Domain detection now produces a weighted
`DomainMix` that fuses four signals (topical embeddings, premise
overlap, ontology entities, and normative markers) without removing or
renaming the existing single-domain `DomainDetector.detect()` API.

- `core/types.py`: added frozen `NormativeProfile(rights, utilitarian,
  deontic)` and frozen `DomainMix(weights, normative,
  ontology_graph_id=None, notes=None, schema_version="domain-mix-v1")`.
  `DomainMix.top()` returns the highest-weight `(domain, weight)` entry
  or `None` for empty mixes. Existing dataclasses are untouched.
- `server/fund/data/normative_markers.json` (NEW): per-axis lemma lists
  under `axes.rights` / `axes.utilitarian` / `axes.deontic`, with
  `schema_version="normative-markers-v1"`.
- `domain/normative.py` (NEW):
  `compute_normative_profile(propositions) -> NormativeProfile`. Uses
  simple regex tokenization on lowercased text, counts token matches
  against each axis's marker list, and returns the share of tokens per
  axis clamped to [0, 1]. Empty input yields an all-zero profile.
  Lexicon load falls back to sane defaults on IO/JSON failures.
- `domain/premises.py`: added module-level `DOMAIN_NORMATIVE` table
  with declared `(rights, utilitarian, deontic)` triples for all 10
  domains, plus `get_domain_normative(key)` that returns
  `(0.33, 0.33, 0.33)` for unknown keys. Existing
  `DOMAINS`/`TENSIONS`/loaders are unchanged.
- `domain/detector.py`: added
  `detect_domain_mix(propositions, ontology, *, embedder=None,
  top_k=3) -> DomainMix`. Legacy `DomainDetector` class is untouched.
  - `w_topic`: cosine similarity between mean proposition embedding
    and per-domain premise centroids (SBERT-or-TF-IDF via
    `embeddings.base.get_embedder` when `embedder is None`).
  - `w_premise`: Jaccard between the argument's content lemmas and
    each domain's premise lemmas after stopword filtering.
  - `w_ontology`: for each domain, normalized count of ontology entity
    surface forms that appear as word-boundary matches inside the
    domain's premise text.
  - `w_normative`: cosine between the argument's `NormativeProfile`
    vector and each domain's declared normative profile.
  - Final per-domain score:
    `0.4*w_topic + 0.25*w_premise + 0.2*w_ontology + 0.15*w_normative`.
  - Graceful degradation: if a signal cannot be computed (missing
    embedder, empty ontology, empty content, empty arg normative
    vector, etc.) it is skipped, its coefficient is redistributed
    across surviving signals, and a `"signal_skipped: <name>"` string
    is appended to the returned `DomainMix.notes` tuple.
  - Sort desc, clamp `top_k` to at least 1, renormalize the retained
    weights so they sum to exactly 1.0. When every signal is skipped
    (e.g. empty propositions and empty ontology), weights is `()` and
    notes lists all four skipped signals.
  - `ontology_graph_id` is a stable sha256-prefix fingerprint of the
    ontology's entities+edges (or `None` for empty/missing ontology).
- `tests/fixtures/hybrid_domain/rights_argument.json` (NEW): 4-
  proposition rights-leaning fixture.
- `tests/fixtures/hybrid_domain/market_argument.json` (NEW): 4-
  proposition market-leaning fixture.
